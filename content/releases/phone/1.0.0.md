# Release 1.0.0

**Initial public prototype**

This is the first exposed version of **Hacking Health App**, focused on validating the core structure, navigation, and data flows. The app is functional as a prototype, with several features scaffolded and others pending backend or wearable integration.

---

## What's included

### Authentication

* Login screen with `username` and `password`.
* No in-app registration flow yet (credentials are assumed to be provisioned externally).

### Home & Core Actions

* Centralized home screen providing access to:

  * Daily Training Session
  * Logs & Documents
  * TxAgent Chat
  * Medical PDF upload
  * Settings & diagnostics
* Navigation and actions are wired and functional at UI level.

### Logs & Documents

* Dedicated Logs screen with internal tabs:

  * Health
  * System
  * Sensor Data
  * Upload Logs
  * Documents
* Tabbed navigation is implemented and ready to receive real data.

### Medical History (PDF Upload)

* Medical documents can be uploaded as PDF files.
* Files are:

  * Stored locally on the device
  * Registered in the local database
* Documents are listed but not yet processed or parsed.

### Settings & Diagnostics

* Connection status with wearable devices.
* Device scanning and binding.
* Base URL configuration.
* Live system log console for debugging and diagnostics.

---

## Known limitations / Not yet implemented

### TxAgent Chat

* Depends on a local backend (`http://10.0.2.2:8000/`).
* Chat history reconstruction is not implemented yet.
* Message persistence is partially scaffolded (explicit TODO remains).

### Document Processing

* Uploaded PDFs are stored and listed only.
* No parsing, analysis, or semantic processing is performed yet.

### Wearable Logs & Uploads

* Health, sensor, and upload logs remain empty until:

  * A real wearable connection is established
  * Data ingestion flows are implemented

---

## Summary

Release **1.0.0** establishes the **foundation** of the application:

* Core screens and navigation
* Local data storage
* Diagnostic and debugging tooling

It is intentionally **incomplete**, serving as a technical baseline for future iterations rather than a production-ready release.
